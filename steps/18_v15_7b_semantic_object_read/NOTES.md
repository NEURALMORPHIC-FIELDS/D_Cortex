# Engineering Notes — Direct Semantic Object Read

## Selection evidence

The Step 15 textual bridge produced accurate semantic query routes but the
trained neural working-memory reader did not turn them into answer uplift.
Step 17 showed that post-hoc fact-value filtering also does not repair the
missing role binding.

The mission explicitly requires token language to become an interface rather
than the substrate of memory operations. Therefore this successor removes the
textual re-route entirely after semantic internalization.

## Epistemic snapshot states

The read-only snapshot represents:

- known entities
- committed `(entity_id, attr_type) -> value_id`
- provisional `(entity_id, attr_type) -> distinct value_ids`

The reader can return:

- committed value
- disputed values
- none object
- none attribute
- refused input

## Explicit non-claims

- no direct execution of sealed Pas 7a
- no write-side semantic ingestion
- no promotion or retrograde
- no mutation of any memory state
- no parser bypass claim outside this isolated object-read contract

## First frozen verdict diagnosis

The direct reader returned zero wrong committed values on known families and
handled every S5 conflict honestly. It nevertheless overcommitted on `13/200`
S6 ambiguous-referent trials.

All thirteen S6 overcommits came from arbitrary approved entity coordinates:

- `crown`: 10
- `scroll`: 3

The pronoun-only query contains neither referent. The next justified safety
layer requires explicit source evidence for the selected entity before an
approved semantic coordinate may reach object memory. This does not address
the still-open F1 coverage gate.
